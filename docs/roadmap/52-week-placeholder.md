# 52-Week Roadmap Placeholder

This roadmap is a planning scaffold, not a locked schedule.

## Phase 1: Foundations (Weeks 1-4)
- Week 1: Product definition, architecture, trust model, repo structure.
- Week 2: Controller skeleton, OpenAI-compatible API skeleton, runtime abstraction, health endpoints.
- Week 3: Policy engine skeleton and first tool execution contracts.
- Week 4: Basic observability and integration test harness.

## Phase 2: Controlled Capability (Weeks 5-12)
- Initial file and process tools with strict policy gates.
- Basic RAG ingestion and retrieval wiring.
- Early user-controlled memory primitives.

## Phase 3: Workflow Depth (Weeks 13-24)
- IDE-centric workflows and coding assistance patterns.
- Service orchestration tools and stronger policy profiles.
- Improved auditability and failure recovery.

## Phase 4: Intelligence and Adaptation (Weeks 25-40)
- Adaptive memory policies and personalization controls.
- Advanced planning heuristics in hybrid controller.
- Optional research mode with network policy controls.

## Phase 5: Portability and Hardening (Weeks 41-52)
- External SSD packaging profile and migration tooling.
- Cross-machine startup reliability and diagnostics.
- Security hardening, performance tuning, and release prep.

## Roadmap Notes
- All phases require explicit security test coverage.
- Offline-first behavior remains default across roadmap stages.
